The goal of my Inventory Tracker app was to give users a simple way to keep track of inventory items. Users can add, view, update, and delete items using a local SQL database. I also included an optional SMS feature that can notify users when an item's quantity is running low. When designing the app, I focused on making the interface easy to use with simple navigation, clear labels, and organized screens so users could quickly complete their tasks.

While building the app, I broke the project into smaller parts and tested each feature before moving on to the next one. This made it much easier to find and fix problems throughout development. Testing also helped ensure that the database, navigation, and user interface all worked together correctly. One of the biggest challenges was connecting every part of the app into one complete application, but by working step by step, I was able to overcome those challenges. The part I am most proud of is successfully connecting the SQL database to the app so users can store, update, and manage inventory items. This project strengthened my programming, testing, and problem-solving skills, and I plan to use these same development strategies in future projects.
